## Detailed Breakdown

### 1. **Python** ğŸ
**Purpose:** Backend server logic and API

**Where it's used:**
- `backend/main.py` - Main FastAPI application (START HERE!)
- `backend/database.py` - Database connection
- `backend/models.py` - Database table definitions
- `backend/schemas.py` - Data validation models
- `backend/auth.py` - Authentication & JWT logic
- `backend/init_db.py` - Database setup script

**What it does:**
- Handles HTTP requests from frontend
- Processes business logic
- Manages database operations
- Provides authentication (login/logout)
- Returns JSON data to frontend

**Files to Run:**
1. `backend/init_db.py` - One-time database setup
2. `backend/main.py` - Start the API server

**Commands:**
```bash
cd backend
python init_db.py    # First time
python main.py        # Every time
```

---

### 2. **JavaScript (React)** âš›ï¸
**Purpose:** Frontend user interface

**Where it's used:**
- `frontend/src/main.jsx` - React entry point
- `frontend/src/App.jsx` - Main app component
- `frontend/src/pages/` - All page components:
  - `Home.jsx` - Landing page
  - `Login.jsx` - Login/Register page
  - `PatientDashboard.jsx` - Patient dashboard
  - `DoctorDashboard.jsx` - Doctor dashboard
  - `AdminDashboard.jsx` - Admin dashboard
- `frontend/src/components/` - Reusable UI components
- `frontend/src/contexts/` - React state management
- `frontend/src/services/` - API calling functions

**What it does:**
- Displays web pages in browser
- Handles user interactions (clicks, forms)
- Makes API calls to backend
- Manages user authentication state
- Updates UI dynamically

**Files to Run:**
- `npm run dev` (runs Vite which uses `vite.config.js`)

**Commands:**
```bash
cd frontend
npm install    # First time
npm run dev    # Every time
```

---

### 3. **SQL / MySQL** ğŸ—„ï¸
**Purpose:** Database storage

**Where it's used:**
- MySQL server (separate application)
- Managed through Python SQLAlchemy in `backend/models.py`

**What it does:**
- Stores all data:
  - Users (patients, doctors, admins)
  - Doctor profiles
  - Appointments
  - Consultations
  - Medical records

**How it's accessed:**
- Python connects via `backend/database.py`
- SQLAlchemy ORM converts Python code to SQL queries
- No direct SQL files - everything is in Python models

**Setup:**
- Install MySQL separately
- Configure credentials in `backend/.env`
- Run `backend/init_db.py` to create database

### **Python Files Run In:**
- Terminal/Command Prompt
- Python Virtual Environment
- Located in `backend/` folder

### **JavaScript Files Run In:**
- Node.js runtime
- Browser (for React rendering)
- Located in `frontend/` folder

### **SQL Executes In:**
- MySQL Server
- Triggered by Python SQLAlchemy
- No direct SQL file editing needed
